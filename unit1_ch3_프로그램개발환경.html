1. 통합 개발 환경 (IDE)
교과서 pp.31–33
IDE의 개념
p.31
통합 개발 환경 (IDE: Integrated Development Environment)
프로그램 개발과 관련한 여러 가지 도구 프로그램, 즉 코드 편집기, 컴파일러/인터프리터, 디버거 등의 프로그램이 유기적으로 연결되어 동작함으로써 편리하고 빠르게 프로그램 개발을 할 수 있도록 돕는 소프트웨어이다.
프로그램 개발 과정: 요구 사항 분석 → 알고리즘 작성 → 소스 코드 작성 → 번역(실행 파일 변환) → 디버깅
소스 코드 (Source Code)
프로그램을 만드는 재료가 되는 코드. 프로그래밍 언어의 문법과 규칙을 사용하여 작성된다.
디버깅 (Debugging)
프로그램에서 발견된 오류를 찾아내고 수정하는 과정이다.
IDE 구성 요소
pp.31–32
① 코드 편집기

프로그램 소스 코드를 작성하고 수정하는 데 사용되는 텍스트 편집 프로그램이다. (p.31)

프로그래밍 언어의 문법과 규칙에 따라 예약어나 기호들을 서로 다른 색깔로 바꾸어 표시함으로써 코드의 가독성을 높여 준다.

들여쓰기를 비롯한 코드 자동 완성 기능을 제공할 뿐만 아니라 원하는 코드 부분을 빠르게 찾거나 바꿀 수 있는 기능도 제공한다.

② 컴파일러 / 인터프리터

소스 코드를 컴퓨터가 실행할 수 있는 형태로 변환하거나 해석하여 실행하는 프로그램이다. (p.32)

구분컴파일러 (Compiler)인터프리터 (Interpreter)
동작 방식 소스 코드 전체를 한 번에 읽어 들인 후 어셈블리어나 기계어로 변환한다. 소스 코드를 한 줄씩 읽고 해석하여 실행한다.
오류 처리 문법적 오류가 있으면 변환하지 못하고 그 이유를 알려 준다. 실행 도중 오류가 발생하면 이유를 알려 주고 실행을 즉시 중단한다.
실행 속도 변환 후 실행하므로 빠르다. 한 줄씩 해석하며 실행하므로 상대적으로 느리다.
해당 언어 C, C++ 등 Python 등
③ 디버거 (Debugger)

오류를 찾아내고 테스트하는 데 사용되는 프로그램이다. (p.32)

프로그램의 실행을 제어하고 실행 과정에서 프로그램이 사용하는 여러 가지 값들과 상태 변화를 추적할 수 있도록 해 준다.

프로그램 실행 도중에 잠시 정지시키거나 원하는 단계까지만 선택적으로 실행시키는 등의 작업이 가능하다.

프로그램이 사용하는 메모리, CPU 사용량도 확인할 수 있어 오류나 잘못된 동작을 찾고 수정하는 데 도움을 준다.

디버거의 주요 기능
실행 제어 — 원하는 지점에서 실행을 멈추거나 단계별로 실행
데이터 추적 — 실행 중 변수 값과 프로그램 상태 변화 확인
오류 판단/수정 — 잘못된 동작의 원인을 파악하고 알고리즘 확인
메모리·CPU 사용량 확인 — 자원 사용 현황 모니터링
IDE의 종류
p.33 [그림Ⅰ-16]

IDE의 종류에 따라 지원하는 프로그래밍 언어, 컴파일러·인터프리터 포함 여부, 코드 강조색, 화면 배치, 클라우드 시스템 연동 여부 등이 다르다. 사용하려는 언어, 개발 방식, 협업 여부, 개인적 익숙함 등을 고려하여 선택한다.

2. 공동 개발 환경
교과서 pp.34–35
공동 개발 환경의 개념
p.34
공동 개발 환경 (온라인)
여러 사람이 협업하여 프로그램을 개발하도록 온라인으로 연결되거나, 특별한 소프트웨어를 설치하지 않고도 웹 브라우저 접속만으로 프로그램을 개발할 수 있는 환경이다.

여러 명의 개발자가 동시에 소스 코드와 문서를 온라인으로 공유·편집하는 과정에서는 버전 관리 및 충돌 문제가 발생할 수 있다. 공동 개발 환경은 이를 해결하기 위한 다양한 기능을 함께 제공한다.

공동 개발 환경의 제공 기능
pp.34–35
기능내용
코드 작성·편집 소스 코드 작성 및 편집, 코드 글자색 강조 표시 기능을 제공한다.
공유 기능 소스 코드 및 문서를 여러 개발자가 온라인으로 공유하며 함께 작업할 수 있다.
버전 관리 변경 사항 추적, 코드 비교, 코드 병합, 히스토리 및 복구 기능을 제공하여 버전 관리 및 충돌 문제를 해결한다.
프로젝트 관리 프로젝트 일정, 작업 분배 및 할당, 작업 상태 추적 등을 관리하는 프로젝트 관리 도구를 제공한다.
메시지·커뮤니케이션 여러 개발자 사이의 원활한 협력 및 의사소통을 위한 자체 메시징 및 커뮤니케이션 서비스를 제공한다.

예) Replit, Eclipse Che, Project Jupyter, GitHub 등 (p.34 [그림Ⅰ-17])

공동 개발 환경의 장점
p.35

시공간 제약 없는 협업: 전 세계 어디서든 팀원들과 온라인으로 협력하여 효율적으로 프로그램을 개발할 수 있다.

개발 효율성 향상: 작업 분배·상태 추적·버전 관리 등의 도구를 통해 여러 개발자의 진행 상황을 파악하며 업무를 조절할 수 있다.

코드 품질 향상: 팀원 간 코드 공유와 상호 검토를 통해 코드의 품질을 높이고 프로젝트를 성공적으로 완료할 수 있다.

글로벌 협력 지원: 전 세계에서 지식을 수집하고 새로운 비즈니스 기회를 창출하는 글로벌 규모의 개발이 가능하다.

통합 개발 환경 vs 공동 개발 환경 비교
pp.31–35
구분통합 개발 환경 (IDE)공동 개발 환경

접속 방법 소프트웨어 설치 필요 (일반적) 웹 브라우저 접속만으로 사용 가능

② 인터프리터는 소스 코드를 한 줄씩 해석하여 실행한다.

④ 인터프리터는 실행 도중 오류가 발생해도 끝까지 실행을 완료한 후 오류를 알려 준다.
⑤ Python은 인터프리터 방식을 사용한다.

문제 4
다음 중 통합 개발 환경(IDE)과 공동 개발 환경에 대한 설명으로 옳은 것은?
① 공동 개발 환경은 반드시 별도의 소프트웨어를 설치해야만 사용할 수 있다.
② IDE는 코드 편집기, 컴파일러/인터프리터, 디버거 등이 유기적으로 연결된 소프트웨어이다.
③ 디버거는 소스 코드를 기계어로 변환하는 역할을 한다.
④ 공동 개발 환경에서는 한 명의 개발자만 파일에 접근할 수 있다.
⑤ 코드 편집기는 프로그램의 오류를 찾아 수정하는 디버깅 기능을 담당한다.

교과서 pp.31–35 참고
IDE는 코드 편집기, 컴파일러/인터프리터, 디버거 등이 유기적으로 연결되어 동작하는 소프트웨어이다. (p.31)

① 공동 개발 환경은 웹 브라우저 접속만으로 사용 가능하다. (p.34)
③ 소스 코드를 기계어로 변환하는 것은 컴파일러의 역할이다. (p.32)

⑤ 오류를 찾아 수정하는 것은 디버거의 역할이다. (p.32)
