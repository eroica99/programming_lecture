1. 산술 연산자
교과서 pp.69–73
연산과 연산자의 개념
p.69
연산 (operation)
미리 정해 놓은 규칙과 방법으로 여러 단계를 거쳐 데이터를 계산하거나 처리하는 것
연산자 (operator)
연산을 표현하는 기호. 연산자를 사용하면 연산 과정을 통하여 결괏값을 만든다.
산술 연산자의 종류
p.69 [표Ⅱ-4]
연산자의미예시결괏값
+합하기2+35
-빼기3-21
*곱하기2*36
/나누기3/21.5
%나눈 나머지3%21
//나눈 몫3//21
**거듭제곱2**38

※ %는 mod(모드) 또는 modulo(모듈로)라고 부르기도 한다.

자료형에 따른 연산 결과
pp.70–71
연산 형태결과 자료형예시결괏값
정수 + 정수정수2 + 35
정수 / 정수실수6 / 32.0
정수 + 실수실수1 + 0.51.5
실수 + 실수실수1.0 + 2.03.0
주의: / 연산은 정수끼리 계산해도 결과가 항상 실수로 만들어진다. 정수 결과가 필요하면 //(몫) 연산자를 사용해야 한다.
또한 실수는 2진수로 변환 시 잘림(truncation) 오차가 발생할 수 있으므로, 정확한 계산을 위해 실수 연산을 최소화하는 것이 좋다.
문자열에 산술 연산자 사용
pp.72–73

+와 * 연산자는 문자열에 사용하면 특별한 의미로 동작한다.

문자열 + 문자열 — 문자열 연결

두 문자열을 이어 붙인 새로운 문자열을 만든다.

코드
s1, s2 = input().split() print(s1 + s2)
입력 → 결과
hello world ↓ helloworld

※ "123" + "456" → "123456" (수 계산이 아닌 문자열 연결)

문자열 * 정수 — 문자열 반복

문자열을 정숫값만큼 반복한 새로운 문자열을 만든다. 정수 * 문자열 순서도 동일하다.

코드
s = input() print(s * 3)
입력 → 결과
hello ↓ hellohellohello

※ "fun" * 3 → "funfunfun"  ·  3 * "fun"도 동일한 결과

2. 비교 연산자
교과서 pp.74–78
비교 연산자의 종류
p.74 [표Ⅱ-5]

두 값의 크기를 비교한 결과를 True 또는 False의 논릿값(불값)으로 반환한다.

연산자의미예시결괏값
<(왼쪽이) 작다2 < 3True
>(왼쪽이) 크다3 > 2True
==(값이 서로) 같다2 == 2True
!=(값이 서로) 다르다2 != 3True
<=(왼쪽이) 작거나 같다2 <= 2True
>=(왼쪽이) 크거나 같다3 >= 2True
주의: 작다(<), 크다(>), 다르다(!)를 =와 함께 쓸 때는 반드시 =가 오른쪽에 와야 한다.
<=, >=, != — 올바름    =<, =>, =! — 오류 발생
문자열 비교
p.78

비교 연산자를 문자열에 사용하면, 가장 앞에 있는 문자부터 순서대로 비교하며 처음으로 달라지는 문자의 유니코드값에 의해 결괏값이 결정된다.

예시결과이유
"abc" == "abc"True모든 문자가 동일
"abc" == "abC"False대문자와 소문자는 다른 유니코드값
"abc" < "abcd"True앞 문자가 같고 길이가 짧을 때 작은 것으로 계산
"abc" > "abC"True소문자의 유니코드값이 대문자보다 크다
3. 논리 연산자
교과서 pp.79–83
논리 연산자의 종류
p.79 [표Ⅱ-6]

여러 가지 복잡한 조건을 간단히 표현할 수 있다. 결과는 항상 True 또는 False의 논릿값이다.

not — 논리 부정 (반대로 바꿈)

주어진 논릿값을 반대로 바꾼다.

논릿값not 결과
TrueFalse
FalseTrue
코드 예시
print(not (123 < 456)) a = int(input()) b = int(input()) print(not (a == b))
결과 (a=123, b=456)
False True
or — 논리합 (하나 이상 True이면 True)

두 논릿값 중 1개 이상이 True이면 True로 계산된다. 둘 다 False일 때만 False이다.

ABA or B
FalseFalseFalse
FalseTrueTrue
TrueFalseTrue
TrueTrueTrue
코드 예시 (2 또는 3의 배수 판별)
n = int(input()) print((n%2==0) or (n%3==0))
입력 5 → 결과
False

※ 5는 2로도 3으로도 나누어떨어지지 않으므로 False or False = False

and — 논리곱 (모두 True이어야 True)

두 논릿값이 모두 True일 때만 True로 계산된다. 하나라도 False이면 False이다.

ABA and B
FalseFalseFalse
FalseTrueFalse
TrueFalseFalse
TrueTrueTrue
코드 예시 (2와 3의 공배수 판별)
n = int(input()) print((n%2==0) and (n%3==0))
입력 6 → 결과
True

※ 6은 2와 3 모두로 나누어떨어지므로 True and True = True (6의 배수)

4. 연산자 우선순위
교과서 p.96 [표Ⅱ-8]

여러 연산자를 소괄호 없이 사용하면, 미리 정해진 우선순위에 따라 계산된다. 우선순위가 높은 연산자가 먼저 계산되고, 우선순위가 같으면 왼쪽에서 오른쪽 순서로 계산된다.

순위연산자설명
높음 ↑**거듭제곱
+x, -x단항 +, 단항 -
*, /, //, %곱, 나눗셈, 몫, 나머지
+, -덧셈, 뺄셈
<, <=, >, >=, !=, ==비교 연산자
not x논리 부정
and논리곱
낮음 ↓or논리합
여러 연산자가 함께 쓰일 때 자신이 생각하는 것과 다르게 계산될 수 있다. 원하는 계산 순서에 따라 소괄호()로 묶어 주는 것이 좋다.
확인 문제
문제 1
다음 산술 연산식의 결괏값을 쓰시오.
연산식결괏값
17 // 5
17 % 5
2 ** 4
10 / 4
"Hi" * 3
정답
교과서 p.69 [표Ⅱ-4] 참고
17 // 5 → 3 (17을 5로 나눈 몫)
17 % 5 → 2 (17을 5로 나눈 나머지)
2 ** 4 → 16 (2의 4제곱)
10 / 4 → 2.5 (/ 연산은 항상 실수 반환)
"Hi" * 3 → HiHiHi (문자열 3회 반복)
문제 2
다음 코드의 실행 결과를 쓰시오.
s1, s2 = input().split() print(s1 + s2) print(s2 + " " + s1)

입력값: Hello World

첫 번째 줄:
두 번째 줄:
정답
교과서 pp.72–73 참고
첫 번째: HelloWorld — s1="Hello", s2="World"이므로 문자열 연결 → 공백 없이 붙음
두 번째: World Hello — s2 + 공백(" ") + s1 순서로 연결
문제 3
다음 비교 연산식의 결괏값(True/False)을 쓰시오.
연산식결괏값
5 >= 5
3 == 3.0
"abc" == "abC"
10 != 10
"abc" < "abcd"
정답
교과서 pp.74–78 [표Ⅱ-5] 참고
5 >= 5 → True (같거나 크다 — 같으므로 True)
3 == 3.0 → True (파이썬에서 정수 3과 실수 3.0은 같다고 계산)
"abc" == "abC" → False (소문자 c와 대문자 C는 유니코드값이 다름)
10 != 10 → False (같은 값이므로 != 결과는 False)
"abc" < "abcd" → True (앞 3글자가 같고 길이가 짧으면 작은 것으로 계산)
문제 4
다음 논리 연산식의 결괏값(True/False)을 쓰시오. (단, n = 6)
연산식결괏값
not (n > 10)
(n%2==0) or (n%5==0)
(n%2==0) and (n%4==0)
not (n%3==0) or (n>5)
정답 (n=6)
교과서 pp.79–83 [표Ⅱ-6] 참고
not (n > 10): 6>10은 False → not False = True
(n%2==0) or (n%5==0): 6%2=0(True) or 6%5=1(False) → True or False = True
(n%2==0) and (n%4==0): 6%2=0(True) and 6%4=2(False) → True and False = False
not (n%3==0) or (n>5): not True or True → False or True = True
문제 5
데이터 연산에 대한 설명으로 옳지 않은 것은?
① 파이썬에서 // 연산자는 나눈 몫을, % 연산자는 나눈 나머지를 계산한다.
② 정수끼리 / 연산을 하면 결과는 항상 실수(float)로 만들어진다.
③ and 연산자는 두 논릿값 중 하나만 True이어도 True로 계산된다.
④ 비교 연산자를 문자열에 사용하면 유니코드값을 기준으로 비교한다.
⑤ 연산자 우선순위에서 ** (거듭제곱)이 +, - (덧셈, 뺄셈)보다 먼저 계산된다.
정답: ③
교과서 p.83 참고
and 연산자는 두 논릿값이 모두 True일 때만 True로 계산된다. 하나라도 False이면 결과는 False이다. 하나 이상의 True로 True가 되는 것은 or 연산자이다.

① //는 몫, %는 나머지 — 옳음 (p.69)
② / 결과는 항상 실수 — 옳음 (p.70)
④ 문자열 비교는 유니코드값 기준 — 옳음 (p.78)
⑤ **의 우선순위가 +, -보다 높음 — 옳음 (p.96)
