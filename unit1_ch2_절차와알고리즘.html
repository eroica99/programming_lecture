① 요구 사항 분석

프로그램에 필요한 요구 사항을 수집 → 분류·조직화 → 기술 → 검증하는 단계이다.

세부 과정내용
요구 사항 수집인터뷰, 설문 조사, 사용자 시나리오 분석 등을 통하여 사용자와 이해관계자로부터 요구 사항을 수집한다.
분류 및 조직화수집된 요구 사항을 기능적 요구 사항과 비기능적 요구 사항으로 분류하고 정리한다.
요구 사항 기술요구 사항을 명확하고 구체적으로 기술한다. 모호한 요구 사항을 명확히 하여 이해관계자 간의 혼란을 최소화한다.
요구 사항 검증사용자와 이해관계자로부터 받은 요구 사항의 정확성과 완전성을 검사·증명하는 과정이다.

요구 사항이 불명확하거나 오류가 있으면 이후 개발 과정 전체에 더 큰 문제가 발생할 수 있으므로 정확한 문서화가 중요하다.

② 설계

요구 사항 분석 결과를 바탕으로 프로그램의 구조를 설계하여 구현에 필요한 구체적 구조와 기능을 검토하는 단계이다.

세부 과정내용
고수준 설계프로그램 구조 설계, 데이터 흐름 다이어그램, 시스템의 구성 요소 및 상호 작용 등을 고려해 전체 구조를 결정한다.
저수준 설계시스템 구성 요소의 동작 방식, 사용할 데이터 구조, 인터페이스 디자인을 결정한다.
설계 검토설계의 완전성, 일관성, 정확성 등을 검증한다. 문제점을 발견하고 수정하여 구현 단계에서 발생할 수 있는 문제를 사전에 방지한다.

설계 단계는 프로그램 개발에 결정적인 역할을 하며, 이후 구현·테스트·유지보수를 효율적으로 가능하게 해 준다.

③ 구현

설계 결과에 따라 프로그램의 목적과 실행 환경에 가장 적합한 프로그래밍 언어를 선택하고 프로그램 코드를 작성하는 단계이다.

표준화한 프로그래밍 방식과 규칙에 따라 작성하여 코드의 품질을 유지하고, 테스트 및 유지보수 단계에서 코드 수정을 쉽게 할 수 있도록 한다.

프로그램 코드에 대한 설명 주석을 충분히 기록하는 것이 좋다.

④ 테스트

구현 단계에서 작성한 프로그램의 정확한 작동 여부를 확인하기 위하여 다양한 시나리오와 데이터를 활용하여 프로그램을 검증하고 오류를 수정하는 단계이다.

단위 테스트(개별 기능 단위 검증)와 시스템 테스트(전체 시스템 검증)를 포함한 여러 유형의 테스트를 진행하며, 발견한 문제점을 수정하여 프로그램의 안정성을 확보한다.

⑤ 유지·보수

프로그램을 성공적으로 개발한 후에도 지속적인 업데이트와 개선 작업이 필요한 단계이다.

주요 작업으로는 오류 수정, 성능 향상, 새로운 기능 적용 등이 있으며, 사용자와 기술적인 변화에 따른 끊임없는 유지보수가 필수적이다.

어떤 문제의 해결을 위한 방법과 절차. 잘 정의된 형식 언어를 사용해서 표현한다.

알고리즘의 특징
p.25 [표Ⅰ-8]
특징설명
입력 작업이나 계산을 위하여 어떤 입력이나 초기 상태로 시작한다.
명확성 구체적이고 명확한 계산이나 처리 방법에 따라 명령을 실행한다. 각 단계에서 수행해야 할 작업이 모호하지 않고 정확해야 한다.
유한성 정확하게 정의되어 있는 유한한 상태들로 바뀐다. 알고리즘은 반드시 일정한 단계 안에 끝나야 하며 무한히 반복되어서는 안 된다.
출력 마지막 상태에서는 작업이나 계산의 결과를 출력하고 종료된다.
알고리즘 표현 방법
pp.26–27
① 순서도 (Flowchart)

특별한 작업이나 처리를 의미하는 여러 가지 도형과 흐름선(화살표)을 사용해서 알고리즘을 표현한 것이다. (p.26 [표Ⅰ-9])

알고리즘의 흐름을 시각적으로 한눈에 파악할 수 있어 복잡한 분기나 반복 구조를 표현하는 데 효과적이다.

컴퓨터로 직접 실행할 수 없으며, 알고리즘을 설계·검토하는 단계에서 주로 사용한다.

② 의사 코드 (Pseudo Code)

일반적으로 사용하는 자연어를 프로그램 코드처럼 사용해서 알고리즘을 표현한 것이다. (p.26)

특정 프로그래밍 언어의 문법을 따르지 않아도 되므로, 어떤 언어로도 구현하기 전에 알고리즘의 논리를 먼저 설계하는 데 사용한다.

컴퓨터로 직접 실행할 수 없다.

③ 프로그래밍 언어

컴퓨터가 실행할 수 있는 프로그래밍 언어를 이용해서 알고리즘을 표현하는 방법이다. (p.27)

특정 프로그래밍 언어의 문법과 규칙에 따라 정확하게 작성해야 하며, 컴퓨터를 통해 직접 실행할 수 있다는 것이 순서도·의사 코드와의 가장 큰 차이이다.
