1. 함수의 개념과 종류
교과서 pp.127–131
함수의 개념
p.127
함수 (function)
어떤 작업의 수행을 위하여 필요한 값과 처리 작업을 작성한 후 새로운 이름으로 정의한 것이다. 프로그래밍 언어에서 함수는 일반적으로 소괄호( ) 기호와 함께 사용된다.
함수는 종류와 환경에 따라 루틴(routine), 서브프로그램(subprogram), 서브루틴(subroutine), 프러시저(procedure), 메서드(method) 등으로 불리기도 한다.
함수를 사용하는 이유
p.127
① 코드 재사용 — 간결한 작성

반복되는 코드를 새로운 이름으로 정의하여 이전보다 간단히 작성할 수 있으며, 크고 복잡한 프로그램을 여러 개의 함수로 나누어 작성할 수 있다.

② 가독성 향상 + 오류 수정 용이

크고 복잡한 프로그램을 여러 개의 함수로 나누어 작성하면 코드의 가독성을 높일 수 있을 뿐만 아니라 프로그램의 실행 단계를 모듈화할 수 있으므로 오류를 쉽게 찾아 수정할 수 있다.

2. 함수의 정의와 호출
교과서 pp.132–134
함수의 정의 (def)
pp.132–133

파이썬에서는 예약어 def를 사용해서 함수를 정의한다. 함수를 정의한 것만으로는 작성 내용이 실행되지 않는다.

형식 요소설명
def함수를 정의하는 예약어. "정의하다(define)"의 줄임말
함수 이름변수 이름을 만드는 규칙과 동일 (영문자·숫자·_만 사용, 숫자로 시작 불가, 대소문자 구분)
소괄호 ( )함수임을 나타내는 기호. 매개변수가 없어도 반드시 작성
콜론 (:)def 줄 끝에 반드시 작성
들여쓰기함수 블록 안의 코드는 반드시 들여쓰기로 작성
함수 정의 (실행 안 됨)
def f(): print("Hello")
정의 + 호출 (실행됨)
def f(): print("Hello") f() # 함수 호출
함수를 정의하는 것을 함수의 선언(declaration)이라고도 한다. 함수를 정의(선언)했을 때 블록 내용이 실행되지 않으며, 반드시 호출(call)해야 실행된다.
함수의 호출 (call)과 복귀 (return)
pp.133–134
함수의 호출 (call)
정의되어 있는 함수의 내용을 불러 실행시키는 것. 함수 이름과 소괄호를 함께 작성하여 호출한다.
복귀 (return)
함수로 정의해 둔 명령이 모두 실행된 후, 함수가 호출되었던 원래 위치로 돌아가 그다음 명령을 실행하는 것이다.
코드
def f(): print("Hello") f() print("World")
실행 흐름
1. f() 호출 2. "Hello" 출력 3. 호출 위치로 복귀 4. "World" 출력
결과
Hello World

※ 함수를 여러 번 호출하면 정의된 코드 블록이 그 횟수만큼 반복 실행된다.

함수 정의 위치 — 반드시 호출보다 앞에
pp.132–133
올바른 순서 (정의 → 호출)
def f(): # 먼저 정의 print("Hello") f() # 그 다음 호출
오류 발생 (호출 → 정의)
f() # 호출 먼저 → 오류! def f(): print("Hello")
파이썬에서 함수는 반드시 정의한 후에 호출해야 한다. 호출 코드가 정의 코드보다 앞에 오면 오류가 발생한다.

교과서 pp.132–133 참고
오류 위치: 1번째 줄 — say_hello() 호출
이유: 함수는 정의(def)보다 호출이 먼저 올 수 없다. 1번 줄에서 say_hello()를 호출할 때 아직 def 블록이 실행되지 않아 함수가 존재하지 않는 상태이다.
수정 방법: def 블록을 호출 코드보다 앞으로 이동
def say_hello(): print("Hello!") say_hello()
