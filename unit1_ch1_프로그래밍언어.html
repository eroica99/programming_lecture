프로그래밍
컴퓨터 프로그램을 만드는 작업. 프로그래밍 언어는 이 작업의 수단이 된다.
프로그램
사람이 원하는 작업을 컴퓨터가 수행하도록 명확하고 구체적인 실행 명령을 순서대로 나열한 것이다. 프로그램을 만드는 코드를 소스 코드(source code)라고 부른다.
프로그래밍 언어
컴퓨터가 실행해야 할 작업을 표현하는 특별한 언어이다. 제한된 기호와 단어를 사용하므로 정확한 형식에 맞게 논리적으로 작성해야 한다.
구분자연어프로그래밍 언어

특징맥락적 의미, 비논리적 표현 포함 가능논리적, 정형화된 기호·단어만 사용
프로그래밍 방식의 분류
pp.14–15
절차적 프로그래밍 (Procedural Programming)

실행해야 할 명령이나 함수를 순서대로 불러 실행하는 방식이다. 변수, 데이터 구조, 함수 등으로 나누어 프로그램을 작성하는 것이 중심이 된다.

지원 언어(예): Fortran, ALGOL, COBOL, BASIC, Pascal, C

객체 지향 프로그래밍 (Object-Oriented Programming)

데이터와 코드를 포함하는 객체(object)를 중심으로 프로그램을 설계·작성하는 방식이다. 객체들 사이의 상호 작용과 변화를 중심으로 이루어진다.

캡슐화, 상속, 다형성 등의 특징 덕분에 코드 재사용이 편리하여 복잡한 프로그램을 효율적으로 개발할 수 있다.

지원 언어(예): C++, C#, Java, Python (C는 객체 지향 불가)

① 기계어 (Machine Language) — 1세대

컴퓨터가 직접 이해하고 실행할 수 있는 언어이다. 0과 1(2진수) 형태로 표현한다. 컴퓨터 하드웨어에는 가장 직관적이지만 인간이 이해하고 작성하기 매우 어렵다.

특징:

· 별도의 번역 과정 없이 CPU가 직접 실행 가능

· 컴퓨터(CPU) 종류마다 다르므로 호환성 없음 — 다른 컴퓨터에서는 같은 명령어가 작동하지 않음

· 실행 속도가 가장 빠르나 작성이 극도로 어려움

예시 (덧셈 명령)
10110000 01100001  →   AL 레지스터에 값 97 저장
② 어셈블리어 (Assembly Language) — 2세대

기계어의 문제점을 해결하기 위해 개발된 언어이다. 기계어 명령과 일대일(1:1) 대응하는 기호(mnemonic)를 사용하여 인간이 한층 쉽게 코드를 작성하고 이해할 수 있도록 만들어졌다.

특징:

· 기계어와 1:1 대응 → 어셈블러(assembler)가 기계어로 변환

· 기계어보다 이해하기 쉽지만 여전히 하드웨어 종속적

· 오늘날에도 시스템 해킹, 컴퓨터 구조 연구, 임베디드 시스템 등에서 사용됨

어셈블리어 예시의미
MOV AX, 5AX 레지스터에 값 5를 저장
ADD AX, 3AX에 3을 더함 (결과: 8)
저급 언어 (Low-Level Language)
기계어와 어셈블리어를 통틀어 저급 언어라고 한다. 컴퓨터 하드웨어에 가까운 언어로, 하드웨어를 직접 제어할 수 있지만 인간이 이해하고 작성하기 어렵다.
고급 언어 (High-Level Language) — 3세대 이상
p.29 읽기 자료 + 보충

1950년대부터 컴퓨터 사용이 늘면서 더 많은 사람이 프로그래밍을 할 수 있도록 인간의 언어에 가까운 고급 언어가 개발되기 시작하였다. Fortran, COBOL이 첫 사례이며, 현재 널리 사용되는 C, Java, Python이 모두 고급 언어에 해당한다.

고급 언어 (High-Level Language)
인간의 언어에 가까운 문법 구조를 가진 프로그래밍 언어이다. 컴파일러 또는 인터프리터를 통해 컴퓨터가 이해할 수 있는 기계어로 번역된다. 가독성이 높고 작성이 쉬워 생산성이 높다.

확인 문제
문제 1
프로그래밍 언어에 대한 설명으로 옳지 않은 것은?
① 프로그래밍 언어는 컴퓨터가 실행해야 할 작업을 표현하는 특별한 언어이다.
② Python은 인터프리터 방식을 사용하기 때문에 실행 속도가 상대적으로 느리다.
③ 기계어는 0과 1의 2진수 형태로 표현하며 컴퓨터가 직접 실행할 수 있다.
④ C는 절차적 프로그래밍과 객체 지향 프로그래밍을 모두 지원한다.
⑤ 고급 언어는 컴파일러 또는 인터프리터를 통해 기계어로 번역된다.
정답: ④
교과서 p.19 [하나 더 알기] 참고
C는 절차적 프로그래밍만 가능하며 객체 지향 프로그래밍은 지원하지 않는다. C, C++, C#, Java, Python 중 객체 지향을 지원하지 않는 언어는 C뿐이다.
문제 2
기계어와 어셈블리어에 대한 설명으로 옳지 않은 것은?
① 기계어는 0과 1의 2진수 형태로 표현되어 CPU가 직접 실행할 수 있다.
② 어셈블리어는 기계어와 일대일(1:1) 대응하는 기호(mnemonic)로 이루어진다.
③ 기계어는 모든 컴퓨터에서 동일하게 실행되는 호환성 높은 언어이다.
④ 어셈블리어는 어셈블러(assembler)에 의해 기계어로 변환된다.
⑤ 기계어와 어셈블리어를 합쳐 저급 언어라고 한다.
정답: ③
교과서 p.29 읽기 자료 참고
기계어는 컴퓨터(CPU)마다 다르기 때문에 호환성이 없다. 다른 컴퓨터에서는 같은 명령어가 작동하지 않는 것이 기계어의 큰 문제점이었다. 이를 해결하기 위해 어셈블리어가 개발되었다.
